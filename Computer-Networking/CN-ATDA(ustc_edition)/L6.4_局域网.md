# 6.4 局域网：编址、ARP、以太网、交换机与 VLAN

---

## 一、ARP 协议：IP 地址与 MAC 地址的桥梁

### 1. IP 地址与 MAC 地址概述

- **IP 地址**（网络层）：
    
    - IPv4 为 32 位，IPv6 为 128 位。
        
    - **分层结构**（网络号 + 主机号），便于路由聚合。
        
    - 作用：用于数据报从源主机到达目标 IP **子网**（前 n-1 跳）。
        
- **MAC 地址**（链路层）：
    
    - 48 位，固化在网卡 ROM 中，**全球唯一**（如 `e9:1A:2F:BB:76:09:AD`）。
        
    - **平面结构**，无层次。
        
    - 作用：用于同一物理网络内节点间的帧传输。
        

### 2. IP 地址的路由作用

- 前 n-1 跳：**网络号**决定路由路径（路由聚合减小路由表）。
    
- 最后一跳：**主机号**定位最终目标主机。
    
- 层次化优势：
    
    - 路由通告时可向上聚合（如将多个子网合并为一个网络号）。
        
    - 向下传递时可细分为具体子网。
        

### 3. MAC 地址的作用

- 物理网络内部唯一标识网卡。
    
- 实现帧的**节点到节点**传输（通过交换机等链路层设备）。
    
- 与 IP 地址的关系：
    
    - IP 是逻辑的、分层的（跨网络路由）。
        
    - MAC 是物理的、平面的（同网络内传输）。
        

### 4. ARP 协议与下一跳

**ARP**（Address Resolution Protocol）的作用：  
将下一跳 IP 地址解析为对应的 MAC 地址，以便封装成帧。

**工作流程**：

1. 路由器根据目标 IP 确定下一跳 IP。
    
2. 通过 ARP 查询下一跳 IP 的 MAC 地址。
    
3. 将分组封装为包含目标 MAC 地址的帧。
    

### 5. 综合传输过程

- **网络层视角**：IP 地址决定路径（n 跳）。
    
- **链路层视角**：每跳通过 MAC 地址完成帧传输（可能经过多台交换机，m 跳）。
    
- **关键**：跨网络时，IP 地址不变，MAC 地址每跳更新。
    

### 6. 网络地址和 MAC 地址分离的优势

|优势|说明|
|---|---|
|**协议兼容性**|链路层可支持多种网络层协议（如 IPX、AppleTalk）。|
|**硬件更换灵活**|网卡损坏时，IP 地址可绑定到新网卡，服务不中断。|
|**过滤效率高**|MAC 地址由硬件直接匹配，不必每个帧都上传 IP 层判断。|

### 7. LAN 地址和 ARP 详解

- **MAC 地址特性**：
    
    - 48 位，全球唯一，厂商前缀由 IEEE 分配。
        
    - 广播地址：`FF-FF-FF-FF-FF-FF`，局域网内所有节点必须接收。
        
- **地址转换过程**：
    
    - 前 n-1 跳：使用下一跳路由器的 IP → 对应 MAC 地址。
        
    - 最后一跳：目标主机的 IP → 目标主机的 MAC 地址。
        
- **移动性支持**：
    
    - MAC 地址是平面结构，网卡可任意移动（如插入不同子网）。
        
    - IP 地址是分层结构，移动后必须改变（除非使用特殊协议）。
        

### 8. ARP 协议工作机制

- **ARP 表**：存储 IP/MAC 映射，每条记录有 **TTL**（典型 20 分钟）。
    
- **查询过程**：
    
    1. 检查 ARP 缓存，若有直接使用。
        
    2. 若无，广播 ARP 请求（目标 MAC 为全 F）：“IP 地址 X.X.X.X 的站点，你的 MAC 地址是什么？”
        
    3. 目标节点单播回复 ARP 响应（包含自己的 MAC）。
        
- **协议特性**：
    
    - **透明**：自动运行，无需配置。
        
    - **即插即用**：自动发现和维护映射。
        
    - **范围**：仅在同一物理网络内有效。
        

### 9. 例题：跨 LAN 通信中的 ARP

**场景**：主机 A（111.111.111.111）→ 路由器 R → 主机 B（222.222.222.222）

- **第一跳**（A → R）：
    
    - 源 MAC：`FF-55`（A 网卡）
        
    - 目标 MAC：`BB-4B`（R 接口）
        
    - 源 IP：`111.111.111.111`
        
    - 目标 IP：`222.222.222.222`
        
- **路由器处理**：解帧 → 查路由表 → 用 ARP 获知 B 的 MAC（`56-2A`）。
    
- **第二跳**（R → B）：
    
    - 源 MAC：`06-9B`（R 出口）
        
    - 目标 MAC：`56-2A`（B 网卡）
        
    - IP 头部不变
        

> **总结**：IP 地址端到端不变，MAC 地址逐跳更新。

## 二、以太网：最主流的局域网技术

### 1. 以太网的发展

- 市场占有率高达 **98%～99%**。
    
- 成本极低（百兆网卡约 30 元，千兆网卡价格持续下降）。
    
- 带宽演进：2M → 10M → 100M（快速以太网）→ 1G（千兆）→ 10G（万兆）。
    
- 数据中心普遍采用 10Gbps 交换机，连接光纤或超五类双绞线。
    

### 2. 以太网的原型

- 受 ALOHA 无线网络协议启发。
    
- Robert Metcalfe 在餐巾纸上绘制了以太网原型。
    
- 最初采用同轴电缆，收发器刺破电缆连接。
    

### 3. 以太网的物理拓扑演变

#### （1）总线型以太网（已淘汰）

- 介质：同轴电缆，两端需**终结器**吸收信号。
    
- 天生广播：任何节点发送，所有节点接收。
    
- 缺陷：电缆破损导致全网瘫痪；CSMA/CD 限制最大距离 2.5 km。
    

#### （2）星型以太网（使用集线器 HUB）

- 物理星型，逻辑总线：一个端口输入，所有端口输出。
    
- **单一冲突域**：同时只允许一个节点成功发送。
    
- 优点：故障隔离（单条线路损坏只影响对应节点）。
    
- 缺点：高负载时 CSMA/CD 效率下降。
    

#### （3）星型以太网（使用交换机）

- **存储转发**：先缓存完整帧，再定向转发。
    
- **端口隔离**：每个端口独立冲突域，支持**全双工**通信。
    
- 多对节点可同时传输，带宽独占。
    

### 4. 以太网帧结构与 CSMA/CD

（原笔记未详细列出帧结构，但后续有 CRC、CSMA/CD 等）

- **不可靠机制**：以太网不发送 ACK/NAK，出错直接丢弃，由上层 TCP 重传（有线介质出错率低）。
    
- **CRC 校验**：硬件实时计算，附加在帧尾，接收方校验，出错则丢弃。
    
- **CSMA/CD 协议**：
    
    - **载波侦听**：发送前先检测信道是否空闲。
        
    - **冲突检测**：发送时持续监听，发现冲突立即停止，发送 Jam 信号强化冲突。
        
    - **二进制指数退避**：
        
        - 首次冲突后等待 `0` 或 `512 bit time`。
            
        - 每次冲突后重传窗口加倍（最大 1024）。
            
        - 10Mbps 网络下，512 bit time ≈ 51.2 μs。
            

### 5. 802.3 以太网标准

- **统一 MAC 协议和帧结构**，速率和物理介质可不同。
    
- 命名规范：如 `10BaseT`：
    
    - `10`：速率 10 Mbps
        
    - `Base`：基带传输
        
    - `T`：双绞线
        
- 最大距离：100 m（双绞线）。
    

### 6. 编码技术

|编码|原理|特点|
|---|---|---|
|**曼彻斯特编码**|每个比特中间有跳变（1: 下跳，0: 上跳）|自带时钟，但效率仅 50%，100M 需 200MHz 带宽|
|**4B5B 编码**|用 5 位编码表示 4 位数据，保证足够跳变|100M 实际传输 125M，带宽增加有限|
|**8B10B 编码**|用 10 位编码表示 8 位数据，用于千兆以太网|支持高速传输，全双工时无需 CSMA/CD|

### 7. 集线器与碰撞域

- **集线器**：物理层中继器，所有端口处于同一**碰撞域**。
    
- 级联仍属于同一碰撞域，高负载时效率急剧下降。
    
- **交换机**：每个端口独立碰撞域，可并发传输。
    

### 8. 交换机工作原理

#### （1）转发表（MAC 地址表）

- 表项：`<MAC 地址， 端口号， 时戳>`
    
- 通过**自学习**动态建立：
    
    - 收到帧时，记录源 MAC 地址和进入端口。
        
    - 时戳用于老化，适应网络变化。
        

#### （2）过滤/转发逻辑

1. **学习**：记录源 MAC 与端口的映射。
    
2. **查表**：用目标 MAC 查表：
    
    - 若表项存在且端口 ≠ 入端口 → **转发**到该端口。
        
    - 若表项存在且端口 = 入端口 → **过滤**（丢弃）。
        
    - 若表项不存在 → **泛洪**（向除入端口外的所有端口转发）。
        

#### （3）自学习示例

- 初始空表：A → A' 的帧触发泛洪。
    
- A' 回复时，交换机记录 A' 的端口。
    
- 后续通信变为单播转发。
    

#### （4）生成树协议（STP）

- 防止环路引起的广播风暴。
    
- 阻塞冗余链路，当主链路故障时自动激活备用链路。

## 三、校园网络拓扑实例（中国科学技术大学）

### 1. 树形分层架构

- **接入层**：终端通过 AP 无线接入或交换机有线接入。
    
- **汇聚层**：楼层交换机 → 大楼核心交换机。
    
- **核心层**：光缆连接各校区核心交换机 → 网络中心。
    

### 2. 可靠性设计

- 东西校区之间配置**备用光纤链路**，平时由生成树协议阻塞。
    
- 主链路中断（如施工挖断），生成树自动激活备用链路。
    

### 3. 服务器与出口

- **服务器集群**（邮件、业务系统）直接连接核心交换机，带宽 10G。
    
- **多 ISP 接入**：通过防火墙连接中国电信、联通、移动、教育网（CERNET）。
    
- 科大网络中心同时作为**安徽教育网核心节点**，通过 SDH 环网连接省内高校。

## 四、交换机 vs 路由器

|对比维度|交换机|路由器|
|---|---|---|
|**工作层次**|数据链路层（L2）|网络层（L3）|
|**转发依据**|MAC 地址|IP 地址|
|**环路处理**|生成树协议（STP）|路由算法（如 OSPF、BGP）|
|**配置**|即插即用，无需配置|需手动配置 IP 地址、子网掩码|
|**广播处理**|转发广播帧（除非 VLAN）|默认不转发广播|
|**性能**|转发速率高（硬件）|相对较低（需查路由表）|

## 五、VLAN：虚拟局域网

- **定义**：通过交换机配置，在物理网络上划分出逻辑上独立的局域网。
    
- **划分方式**：
    
    - 基于端口（最常用）
        
    - 基于 MAC 地址
        
    - 基于协议
        
- **特性**：
    
    - 广播帧仅在 VLAN 内转发，实现**逻辑隔离**。
        
    - 可跨多台交换机（需配置 Trunk）。
        
    - 提高安全性、灵活性，简化网络管理。

## 六、知识小结

|知识点|核心内容|考试重点/易混淆点|难度|
|---|---|---|---|
|**ARP 协议**|IP → MAC 转换，局域网内有效|IP 分层 vs MAC 平面|★★★★|
|**IP 与 MAC 区别**|IP 分层用于路由，MAC 平面用于物理传输|最后一跳才用到主机号|★★★★★|
|**以太网演进**|总线 → 集线器 → 交换机|CSMA/CD 在高负载下的效率|★★★|
|**交换机原理**|MAC 表自学习，过滤/转发，泛洪|自学习 vs 路由表|★★★★★|
|**VLAN**|逻辑隔离广播域|基于端口划分|★★★★|
|**生成树协议**|防止环路，主备切换|冗余链路设计|★★★★|
|**曼彻斯特编码**|自带时钟，效率 50%|4B5B/8B10B 对比|★★★|
|**校园网拓扑**|核心-汇聚-接入三层结构|生成树实际应用|★★★★|
|**交换机 vs 路由器**|L2 vs L3，转发依据不同|环路处理差异|★★★★★|

---

> **核心启示**：局域网是互联网的“最后一公里”。从 MAC 地址到 ARP，从以太网到交换机，再到 VLAN 和生成树，每一个机制都是为了在共享介质上实现高效、可靠、可管理的通信。理解它们，就掌握了现代网络的基础构建块。
