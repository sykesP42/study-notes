## 一、互联网基础概念

### 1. 互联网的构成

- **节点**：端系统（主机+应用） + 路由器
    
- **边**：通信链路
    
- **本质**：网络的网络（network of networks）
    

### 2. 网络边缘

- **端系统**：运行应用的设备
    
- **交互模式**：
    
    - **C/S模式**：服务器提供服务，客户端请求服务（集中式）
        
    - **P2P模式**：节点既是客户端又是服务器（去中心化）
        

### 3. 网络核心

- **核心功能**：数据交换（存储转发）
    
- **工作方式**：分组交换（基于统计多路复用）
    
- **交换技术对比**：
    
    - **虚电路**：需建立网络层连接（如ATM）
        
    - **数据报**：分组独立路由（如IP）
        

### 4. 接入网与物理介质

- **接入类型**：住宅接入（xDSL、HFC）、机构接入（以太网/WLAN）
    
- **物理介质**：
    
    - **导向介质**：双绞线、同轴电缆、光纤
        
    - **非导向介质**：无线电磁波

## 二、分组交换网络性能

### 1. 四类延迟

|延迟类型|计算公式|影响因素|
|---|---|---|
|处理延迟|固定|检查首部、路由决策|
|排队延迟|可变|网络拥塞程度|
|传输延迟|L/RL/R|分组长度LL，链路速率RR|
|传播延迟|d/sd/s|距离dd，信号速度ss|

### 2. 流量强度与丢包

- **流量强度** I=La/R （a：分组到达率）
    
- I→1 时排队延迟→∞，缓冲区溢出导致丢包
    

### 3. 吞吐量

- **瓶颈链路**：端到端吞吐量 = min⁡(R1,R2,…,Rn)
    
- 每跳共享时：单个连接获得 R/n

## 三、协议层次与分层原理

### 1. 五层模型

|层次|功能|典型协议|
|---|---|---|
|应用层|应用报文交换|HTTP、FTP、SMTP、DNS|
|传输层|进程间通信|TCP、UDP|
|网络层|路由转发|IP、ICMP、路由协议|
|链路层|帧传输|以太网、PPP|
|物理层|比特流传输|曼彻斯特编码、4B5B|

### 2. 封装过程

- 发送端：自上而下添加首部
    
- 路由器：网络层解封装 + 路由决策
    
- 接收端：自下而上剥离首部

## 四、应用层核心协议

### 1. 电子邮件

- **系统构成**：用户代理 + 邮件服务器 + 协议
    
- **传输协议**：SMTP（推模式）
    
- **存取协议**：POP3 / IMAP（拉模式）
    
- **报文格式**：首部（发件人、收件人等）+ 报文体
    
- **MIME扩展**：支持多媒体内容编解码
    

### 2. DNS系统

- **核心作用**：域名 → IP 地址、别名 → 规范名、邮件服务器解析
    
- **特性**：应用层基础设施、分布式数据库、区域划分
    
- **解析方式**：递归查询 vs 迭代查询
    
- **缓存机制**：缓存记录有TTL，权威记录永久保存
    

### 3. P2P网络

- **核心优势**：可扩展性强（服务能力随节点增加而增长）、分发效率高
    
- **典型应用**：Napster、Gnutella、BT、DHT
    

### 4. 视频流化与CDN

- **流化方式**：UDP流化、HTTP流化、DASH（动态自适应）
    
- **DASH流程**：获取manifest → 根据带宽/缓冲动态请求不同质量块
    
- **CDN技术**：
    
    - **背景**：解决单服务器扩展性问题
        
    - **工作原理**：部署缓存节点 + 内容预部署 + DNS重定向
        
    - **部署策略**：Enter Deep（深入ISP）、Bring Home（靠近骨干）

## 五、传输层详解

### 1. TCP vs UDP

|特性|TCP|UDP|
|---|---|---|
|连接性|面向连接|无连接|
|可靠性|可靠（确认、重传、序号）|不可靠|
|数据边界|字节流（无边界）|保留报文边界|
|首部开销|20-60字节|8字节|
|适用场景|文件传输、网页浏览|实时应用、DNS|

### 2. Socket编程

- **TCP Socket**：四元组（源IP、源端口、目的IP、目的端口）标识连接
    
- **UDP Socket**：二元组（目的IP、目的端口）标识端点
    
- **三次握手**：SYN → SYN+ACK → ACK
    
- **四次挥手**：FIN → ACK → FIN → ACK（两军问题）
    

### 3. 可靠数据传输原理（RDT）

| 协议      | 机制            | 特点          |
| ------- | ------------- | ----------- |
| RDT 1.0 | 理想信道          | 无差错、无丢失     |
| RDT 2.0 | 校验和 + ACK/NAK | 处理比特错误      |
| RDT 2.1 | 序号            | 解决ACK/NAK出错 |
| RDT 2.2 | 无NAK          | 用编号ACK替代    |
| RDT 3.0 | 超时重传          | 处理丢包        |

### 4. 滑动窗口协议

|协议|发送窗口|接收窗口|重传粒度|
|---|---|---|---|
|停等|1|1|单包|
|Go-Back-N|>1|1|批量重传|
|Selective Repeat|>1|>1|选择性重传|

### 5. TCP拥塞控制

- **AIMD**：加性增（每RTT +1 MSS）、乘性减（丢包时减半）
    
- **慢启动**：指数增长，阈值ssthresh
    
- **拥塞避免**：线性增长
    
- **快速重传**：收到3个冗余ACK立即重传
    
- **快速恢复**：Reno算法，避免回到慢启动

## 六、网络层

### 1. 数据平面 vs 控制平面

|平面|功能|传统方式|SDN方式|
|---|---|---|---|
|数据平面|分组转发|查路由表|匹配流表|
|控制平面|路由计算|分布式协议（OSPF、BGP）|集中控制器|

### 2. IP协议

- **尽力而为服务**：可能丢包、乱序、不可靠
    
- **IP数据报格式**：版本、首部长度、总长度、标识、标志、片偏移、TTL、协议、校验和、源/目的IP
    
- **分片与重组**：
    
    - 分片原因：超过MTU
        
    - 重组位置：目标主机
        
    - 偏移量单位：8字节
        

### 3. IP编址

- **子网**：相同前缀 + 一跳可达
    
- **地址分类**：
    
    - A/B/C类：单播
        
    - D类：多播
        
    - E类：预留
        
- **特殊地址**：内网地址（10/8、172.16/12、192.168/16）、回路地址（127/8）
    
- **CIDR**：无类域间路由，任意位划分网络号
    
- **DHCP**：自动获取IP、掩码、网关、DNS
    

### 4. IPv4 → IPv6

- **主要变化**：地址128位，头部简化
    
- **过渡技术**：隧道技术
    

### 5. 路由协议

|协议|算法|范围|特点|
|---|---|---|---|
|RIP|距离矢量|内部|简单，跳数限制15|
|OSPF|链路状态|内部|快速收敛，支持分层|
|BGP|路径矢量|外部|策略控制，AS_PATH防环|

## 七、链路层

### 1. 服务与连接方式

- **服务**：成帧、链路接入、可靠交付、差错检测/纠正
    
- **连接类型**：
    
    - **点对点链路**：独占连接（如PPP）
        
    - **多点连接**：共享信道（如以太网）
        

### 2. 差错控制：CRC

- 原理：模2除法，生成多项式 GG
    
- 发送：D⋅2r⊕R，R=remainder(D⋅2r/G)
    
- 接收：用 GG 除，余数为0则无错
    

### 3. MAC协议

|协议|原理|应用|
|---|---|---|
|CSMA/CD|边听边说，冲突后退避|以太网|
|CSMA/CA|冲突避免，ACK确认|802.11无线|

### 4. 以太网与交换机

- **MAC地址**：48位，全球唯一
    
- **ARP协议**：IP → MAC 映射（广播查询）
    
- **交换机**：
    
    - 自学习：记录源MAC与端口
        
    - 转发决策：查表→转发/泛洪/过滤
        
    - 生成树协议：防止环路
        

### 5. 以太网物理拓扑

- 总线型 → 集线器 → 交换机
    
- 编码：曼彻斯特、4B5B、8B10B

## 八、网络安全

### 1. 四大要素

- **机密性**：加密防止窃听
    
- **可认证性**：验证身份
    
- **报文完整性**：防止篡改
    
- **可用性**：服务持续可用
    

### 2. 加密技术

|类型|特点|算法|
|---|---|---|
|对称加密|加解密同一密钥|AES、DES|
|非对称加密|公钥+私钥|RSA|

### 3. 数字签名

- **实现**：用私钥加密报文摘要
    
- **特性**：可验证、不可伪造、不可抵赖
    
- **摘要算法**：MD5、SHA（需防碰撞）
    

### 4. 安全设备

- **防火墙**：包过滤、有状态检测
    
- **IDS/IPS**：入侵检测/防御系统

## 九、未来网络技术

### 1. 软件定义网络（SDN）

- **核心思想**：控制与数据平面分离
    
- **三要素**：开放转发硬件接口、统一控制平台、北向API
    
- **OpenFlow**：南向接口协议，流表（规则+动作+统计）
    

### 2. 命名数据网络（NDN）

- **核心思想**：以内容为中心，基于名称路由
    
- **报文类型**：Interest（请求）、Data（响应）
    
- **关键数据结构**：PIT（待定兴趣表）、FIB（转发信息表）、CS（内容存储）
    
- **优势**：网内缓存、请求聚合、安全内建
    

### 3. 移动优先网络（MobilityFirst）

- **核心思想**：名字（GUID）与地址（NA）分离
    
- **GNRS**：全局名字解析服务（INSERT/UPDATE/LOOKUP）
    
- **抗毁路由**：网内存储 + 动态重路由
    

### 4. 网络试验设施：CENI

- **规模**：40城市、88主干节点、133边缘节点、100G带宽
    
- **分中心**：南京（可编程虚拟）、北京（IPv6）、合肥（智能网络）、深圳（云化边缘）
